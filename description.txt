1. Создан favicon, как на настоящем сайте Fitness House
2. Страница № 1 полностью адаптивна
3. Данные из файла JSON загружаются из Firebase. Если при первой попытке соединения с БД не удалось загрузить данные, то через 3 сек. происходит повторное соединение с БД и так до тех пор, пока с БД не удастся соеденится
4. Картинки перенесены в Storage Firebase, файл JSON перенесен в RealTime DataBase. После перехода на главную страницу происходит загрузка из Firebase
5. Для фильтров формируется отдельный массив
6. Для более привлекательного отображения на мобильных устрйоствах, фильтр вместо <select> заменен на обычный список, при этом основные возможности <select> сохранены:
	- при клике на название фильтра либо на кнопку в виде галочки, открывается контекстное меню
	- при клике в любом месте страницы, в т.ч. и на другие поля фильтра, фильтр, который был открыт - автоматически закрывается
	- при повторном клике на название фильтра либо на кнопку в виде галочки, контекстное меню закрывается
	- при выборе пункта контекстного меню, данный пункт отображается вместо названия фильтра, контекстное меню автоматически закрывается
